# 개념: Point · Segment · Face · Region

poly-forge는 DynEarthSol3D(DES3D)용 2D `.poly`(PSLG) 메시 입력 파일을 만드는 에디터입니다.
이 문서는 화면 통계(Statistics)에 나오는 **Point / Segment / Face / Region**의 뜻과 차이,
그리고 서로 어떻게 연결되는지를 설명합니다.

> 좌표계: `x`는 수평, `z`는 깊이입니다(`z ≤ 0`, 아래로 내려갈수록 작아짐). 단위는 meter.

---

## 한눈에 보기

| 항목 | 차원 | 정체 | `.poly`에 저장? |
|---|---|---|---|
| **Point** | 0D (점) | 꼭짓점 좌표 `(x, z)` = node | ✅ node 섹션 |
| **Segment** | 1D (선) | 두 점을 잇는 선분(변/edge) | ✅ segment 섹션 |
| **Face** | 2D (면) | 선분들이 **둘러싼 닫힌 영역** (앱이 계산) | ❌ 저장 안 함 |
| **Region** | 0D (씨앗점) | 면에 **물성을 부여한 결과**(seed + mattype + size) | ✅ region 섹션 |

핵심: **Segment는 그린 선, Face는 그 선들이 만든 빈 칸(앱이 계산, 파일엔 없음),
Region은 그 칸에 "재료·요소크기"를 도장 찍은 씨앗점(파일에 저장, DES3D가 실제로 사용).**

---

## Point (점)

- 꼭짓점입니다. `.poly`의 node 섹션에 `i x z`로 저장됩니다(id는 0부터 연속).
- 그리드 클릭, 또는 컨트롤 패널의 *Add point* 로 추가합니다. 같은 좌표는 중복 생성되지 않고 기존 점을 재사용합니다.

## Segment (선분)

- 두 점을 잇는 **선**입니다. `.poly`의 segment 섹션에 `j p0 p1 bdry_flag`로 저장됩니다.
- `bdry_flag`(경계 플래그)는 **단일 비트만** 허용됩니다:
  - `0` = 내부(점선으로 표시)
  - `1` = 좌(X0), `2` = 우(X1), `16` = 하(Z0), `32` = 상(Z1) — 각각 다른 색으로 표시
- 외곽 경계선은 도메인 가장자리에 놓이며, *Bulk edit → Auto-assign boundary flags* 로 자동 부여할 수 있습니다.

→ **"무엇이 선으로 연결돼 있는가"** 를 나타냅니다.

## Face (면)

- 선분들이 닫힌 고리를 이루면, 그 **안쪽 빈 공간**을 앱이 자동으로 계산해 Face로 인식합니다
  (내부적으로 `@turf/polygonize` 사용).
- **`.poly` 파일에는 들어가지 않습니다.** 순수하게 *"여기 둘러싸인 영역이 하나 생겼다"* 는
  기하학적 사실이며, 사용자가 클릭해 물성을 줄 수 있게 해주는 **편의 장치**입니다.
- 면의 채움색 = 그 면 안에 들어있는 **Region의 mattype 색**. 재료가 지정되기 전에는 **회색**입니다.

→ **"닫힌 영역이 몇 개 인식됐는가"**.

### 면 분할과 noding (T-junction)

면을 둘로 나누려면 그 면의 경계 위 두 점을 잇는 선을 그립니다. 이때 새 점이 기존 변의
**중간**에 놓이면, polygonize가 인식할 수 있도록 **그 변을 새 점에서 자동으로 쪼갭니다(noding)**.
또한 선/점 도구에서는 커서가 변 근처에 있으면 **변 위로 스냅**되어, 비스듬한 경계에도
정확히 점을 올려놓고 면을 나눌 수 있습니다.

- 점 배치 스냅 우선순위: **기존 꼭짓점 → 변 위 투영점 → 그리드 교차점**.

## Region (영역)

- DES3D가 실제로 이해하는 개념입니다. 면 목록이 아니라
  **영역 내부의 점 하나(씨앗점, `xk·zk`) + 물성번호(`mattype`, 0부터) + 최대 요소 크기(`size`)** 입니다.
- `.poly`의 region 섹션에 `k xk zk mattype size`로 저장됩니다. `size > 0`은 최대 면적 제한,
  `size = -1`은 무제한입니다.
- DES3D/Triangle은 이 씨앗점에서 바깥으로 채워나가며 *"이 칸은 mattype N 재료, 요소 크기는 X"* 로
  메시를 생성합니다.
- **DES3D는 region이 1개 이상(`nregions ≥ 1`)이어야 합니다.** region이 하나도 없으면
  내보낸 파일을 메시화할 수 없습니다(검증 시 에러로 표시됩니다).

→ **"각 칸이 어떤 재료/해상도인가"**.

---

## Face ↔ Region: 가장 중요한 차이

- **Face = 기하(껍데기)**: 그냥 닫힌 빈 공간. 재료 정보 없음.
- **Region = 물성(알맹이)**: 그 공간에 "재료 X"라고 도장 찍은 씨앗점.

둘은 보통 1:1로 대응하지만 **달라질 수 있습니다.**

### 면을 Region으로 만들기

1. 선택 도구로 면을 클릭(또는 `Ctrl` + 드래그로 다중 선택)합니다.
2. Inspector에서 **mattype / size** 를 입력합니다.
3. 그러면 면 중심에 region 씨앗점이 하나 찍히고, 면이 그 재료색으로 칠해집니다.
4. 모든 면을 한 번에 채우려면 *Bulk edit → Fill all faces with mattype* 을 사용합니다.

### 고아(orphan) Region

Region은 면과 **독립적인 점**입니다(DES3D 모델이 그렇습니다). 그래서 면을 만든 선을 지우면
면은 사라지지만 **region 씨앗점은 남습니다.** 어떤 닫힌 면에도 속하지 않게 된 region을
"고아(orphan)"라고 부르며, DES3D에서는 잘못 메시되므로 다음과 같이 다룹니다.

- 캔버스에서 **빨간 점선 테두리**로 표시됩니다.
- *Validate* 가 경고로 알려줍니다.
- *Bulk edit → Remove orphaned regions* 로 한 번에 정리할 수 있습니다.

> 설계 선택: 지오메트리를 편집할 때 region을 **자동으로 삭제하지 않습니다**(사용자가 입력한
> 재료 정보를 말없이 잃지 않도록). 대신 눈에 띄게 표시하고, 원할 때 직접 정리하도록 합니다.
> (편집 중 자동 삭제를 원하면 동작을 바꿀 수 있습니다.)

## `.poly` 파일과의 매핑

```
#### node coordinates ####     <- Points
# npoints ndims 0 0
13 2 0 0
0 0 0
...
#### segments ####             <- Segments
# nsegments has_bdryflag
16 1
0 0 1 1                        # j p0 p1 bdry_flag
...
#### holes ####                <- 항상 0 (DES3D는 hole 미지원)
0
#### regions ####              <- Regions (씨앗점)
4
0 250000 -20000 0 2e7          # k xk zk mattype size
...
```

**Face는 이 파일 어디에도 없습니다** — 앱이 화면에서만 계산하는 값입니다.

---

## 예시: rifting-2d 샘플 (13 / 16 / 4 / 4)

- **13 Points**: 단면의 꼭짓점들
- **16 Segments**: 바깥 테두리(좌=파랑, 우=빨강, 상=초록, 하=주황) + 지층을 나누는 내부 선(점선)
- **4 Faces**: 그 선들이 만든 4칸의 닫힌 영역(상부층 1칸 + 하부층 3칸)
- **4 Regions**: 각 칸 안의 씨앗점 4개 — 상부 1칸은 mattype 0(파랑), 하부 3칸은 mattype 1(빨강).
  **이것이 실제로 메시 재료를 결정하는 데이터**입니다.

이 면들 중 하나를 선으로 다시 나누면 Faces가 5로 늘어나고, 새로 생긴 면은
아직 region이 없으므로 회색으로 표시됩니다 — 거기에 재료를 지정하면 Region이 하나 더 생깁니다.
